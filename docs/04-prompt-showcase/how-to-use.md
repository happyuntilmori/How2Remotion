# 쇼케이스 활용법

## remotion.dev/prompts 탐색

브라우저에서 [https://www.remotion.dev/prompts](https://www.remotion.dev/prompts)를 열면 갤러리 형태로 영상 예제들이 표시됩니다.

---

## 화면 구성 이해

각 예제 카드에는 다음 정보가 표시됩니다.

- **영상 썸네일**: 마우스를 올리면 미리보기가 재생됩니다
- **프롬프트 텍스트**: 해당 영상을 생성하는 데 사용된 실제 프롬프트
- **컴포지션 정보**: 크기(width × height), fps, 총 길이

예제를 클릭하면 상세 페이지로 이동합니다. 상세 페이지에서는 전체 프롬프트와 소스 코드를 볼 수 있습니다.

---

## 예제 카테고리

쇼케이스의 예제들은 크게 다음 카테고리로 분류됩니다.

### 텍스트 애니메이션
단어, 문장, 헤드라인이 다양한 방식으로 나타나는 예제입니다. 자막, 타이틀 카드, 슬로건 영상에 적합합니다.

대표 예제:
- 단어가 하나씩 팝업되는 효과
- 글자가 흩어졌다 모이는 효과
- 카운터 숫자 애니메이션

### 제품 데모
SaaS 제품이나 앱의 기능을 소개하는 영상입니다. 목업 화면, 기능 목록, 비교 차트 등을 포함합니다.

대표 예제:
- 앱 화면이 등장하는 제품 소개
- 기능 3가지를 순서대로 소개하는 영상
- "전/후" 비교 영상

### 데이터 시각화
숫자나 통계를 애니메이션으로 표현하는 예제입니다. 차트, 그래프, 진행 바 등을 포함합니다.

대표 예제:
- 막대 그래프 성장 애니메이션
- 원형 진행률 표시
- 숫자가 증가하는 카운터

### 소셜 미디어 콘텐츠
Instagram Reels, YouTube Shorts 등 세로형(9:16) 콘텐츠 예제입니다.

대표 예제:
- 짧은 팁 영상 (30초)
- 제품 광고 클립
- 이벤트 공지 영상

## 원하는 예제 찾는 방법

내가 만들려는 영상에 가장 가까운 예제를 찾는 단계입니다.

**1단계: 비슷한 썸네일 찾기**

갤러리를 스크롤하면서 레이아웃, 색상, 움직임이 내가 원하는 것과 비슷한 예제를 찾습니다. 완벽히 일치할 필요는 없습니다. 60~70% 비슷하면 충분합니다.

**2단계: 프롬프트 읽기**

해당 예제의 프롬프트를 읽어봅니다. 어떤 정보를 어떤 순서로 기술했는지 파악합니다.

**3단계: 소스 코드 확인**

코드를 열어봅니다. 사용된 Remotion 컴포넌트와 훅을 확인합니다. 코드를 전부 이해할 필요는 없고, 어떤 구조인지만 파악합니다.

**4단계: 다운로드 결정**

- 내가 원하는 것과 비슷하다 → 다운로드 후 커스터마이징
- 다르지만 일부 효과만 참고하고 싶다 → 프롬프트의 해당 부분만 차용

## 예제 메타데이터 읽는 법

각 예제의 상세 페이지에는 다음과 같은 메타데이터가 표시됩니다.

```
Composition: ProductIntro
Size: 1920 × 1080
FPS: 30
Duration: 5.0s (150 frames)
```

이 정보는 프로젝트를 시작할 때 `Root.tsx`의 `<Composition>` props에 그대로 입력합니다.

```tsx
<Composition
  id="ProductIntro"
  component={ProductIntro}
  durationInFrames={150}
  fps={30}
  width={1920}
  height={1080}
/>
```

## 어떤 예제가 내게 맞는지 판단 기준

| 목표 | 적합한 카테고리 | 확인할 것 |
|------|----------------|-----------|
| 제품 인트로 영상 | 제품 데모 | 씬 전환 방식 |

| 발표 자료 강조 | 텍스트 애니메이션 | 텍스트 등장 속도 |
| 실적 보고 영상 | 데이터 시각화 | 차트 유형 |
