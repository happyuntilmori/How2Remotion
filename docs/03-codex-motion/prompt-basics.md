# 프롬프트 작성법

## 효과적인 Codex 프롬프트의 구조

Codex에게 Remotion 코드를 요청할 때, 모호한 요청은 모호한 코드를 낳습니다. 다음 네 가지 요소를 포함하면 원하는 결과를 훨씬 빠르게 얻을 수 있습니다.

```
[영상 사양] + [비주얼 씬 설명] + [스타일] + [오디오/텍스트 내용]
```

---

## 나쁜 프롬프트 vs 좋은 프롬프트

### 나쁜 프롬프트 예시

```
"멋진 텍스트 애니메이션 만들어줘"
```

무엇이 문제인가:
- 영상 크기가 없음 (1920×1080? 9:16?)
- "멋진"이 무엇인지 정의되지 않음
- 텍스트 내용이 없음
- 애니메이션 방향과 타이밍이 없음

Codex가 임의로 결정해야 하는 항목이 많아지면 원하는 결과가 나올 확률이 낮아집니다.

### 좋은 프롬프트 예시

```
1920x1080 영상을 만들어줘. 배경은 짙은 남색(#1a1a2e).
텍스트 "TechFlow — 업무를 자동화하다"가
프레임 0에서 아래 40px 위치에서 시작해
프레임 20까지 제자리로 올라오면서 페이드인.
폰트는 bold, 크기 72px, 색상 흰색.
총 길이 90프레임(30fps 기준 3초).
```

좋은 이유:
- 영상 사양 명시 (1920×1080, 30fps, 90프레임)
- 정확한 색상 코드 사용
- 애니메이션 시작 위치, 끝 위치, 타이밍 명시
- 텍스트 내용 포함

## 프롬프트 구조 템플릿

다음 템플릿을 복사해 사용합니다.

```
[영상 사양]
크기: {너비}x{높이}
fps: {fps}
총 길이: {프레임 수}프레임 ({초}초)

[비주얼 설명]
배경: {색상 또는 그라디언트}
요소 1: {설명, 위치, 크기}
요소 2: {설명, 위치, 크기}

[애니메이션]
- {요소 1}: 프레임 {시작}~{끝} 동안 {동작}
- {요소 2}: 프레임 {시작}~{끝} 동안 {동작}

[텍스트 내용]
{실제 텍스트}

[반드시 지켜야 할 규칙]
- CSS @keyframes 사용 금지, Remotion의 interpolate() 사용
- extrapolateRight: 'clamp' 항상 포함
- named export로 MyComposition 내보내기
```

---

## Codex에게 반드시 상기시켜야 할 규칙

다음 규칙들은 스킬 모듈이 있어도 복잡한 프롬프트에서 Codex가 놓칠 수 있습니다. 중요한 요청에는 명시적으로 포함하는 것이 좋습니다.

| 규칙 | 이유 |
|------|------|
| `CSS @keyframes` 금지 | Remotion 렌더러에서 동작하지 않음 |
| `interpolate()`에 `extrapolateRight: 'clamp'` | 범위 초과 시 값 폭주 방지 |
| `useCurrentFrame()`으로만 프레임 접근 | 다른 방법은 렌더링 시 오작동 |
| `named export`로 컴포넌트 내보내기 | Root.tsx에서 import하려면 필요 |
| `useVideoConfig()`에서 fps 가져오기 | spring() 계산에 반드시 필요 |

## 반복적 프롬프트 (Iterative Prompting)

한 번에 완벽한 결과를 기대하기보다, 여러 번의 대화로 점진적으로 개선하는 것이 효율적입니다.

**1단계: 기본 구조 생성**

```
1920x1080, 30fps, 90프레임 영상.
짙은 남색 배경에 흰 텍스트 "TechFlow"가 페이드인되는 컴포지션 만들어줘.
```

**2단계: 세부 조정**

```
텍스트 아래에 '#0066CC' 색상의 200px 너비 밑줄이
텍스트보다 10프레임 늦게 왼쪽에서 오른쪽으로 확장되도록 추가해줘.
```

**3단계: 타이밍 조정**

```
밑줄 애니메이션이 너무 빠르다. 20프레임 동안 천천히 나타나도록 조정해줘.
```

각 단계에서 Studio로 결과를 확인하고, 다음 피드백을 줍니다. 이 방식이 한 번에 모든 것을 설명하려는 것보다 훨씬 안정적인 결과를 낳습니다.
